# 一、项目与岗位匹配诊断

## 1.1 项目与岗位匹配度：62 / 100（偏技术，产品侧偏弱）

| 维度 | 匹配情况 | 说明 |
| --- | --- | --- |
| **AI 应用落地** | ⭐⭐⭐⭐ | DeepSeek 结构化输出、SSE、Engine + LLM 分层 — 证明「会用 AI 做产品」 |
| **跨境电商场景** | ⭐⭐⭐ | Coquette / 选品 / 趋势 / 预警 — 场景对，但数据是 Mock，缺 TikTok Shop 真实链路 |
| **产品方法论** | ⭐⭐ | 有功能矩阵，缺用户、痛点验证、PRD、优先级、指标 |
| **助理岗核心活** | ⭐⭐ | 调研 / 访谈 / 竞品 / 原型 / Coze·Dify 工作流 — 简历和作品里几乎没体现 |
| **工程协作** | ⭐⭐⭐⭐ | Next.js 全栈 + 41 测试 — 加分，但别包装成「前端开发实习」 |

**结论：** TrendScout 是**合格的 AI 应用 Demo + 技术作品集**，对「懂 AI 的产品助理」是**敲门砖**，不是**录取通知书**。面试官会认可你「能和技术对话」，但会质疑你「是不是只会堆页面」。

---

## 1.2 当前能力短板

### 产品侧（主短板）

- 说不清 **目标用户是谁、谁付钱、为什么现在就要用**
- 没有 **用户调研 / 访谈记录 / 竞品对比表 / PRD** 等可展示产物
- 功能命名与真实能力不一致（部分「AI」实为规则引擎）— 产品 sense 会被追问
- 无 **核心指标**（如：趋势发现耗时、选品决策时间、预警准确率）

### AI 产品侧

- 会调 API，但说不清 **Prompt 版本、Fallback 策略、成本、延迟、幻觉控制**
- 未涉及 **RAG / Agent / Dify / Coze** 等工作流平台 — 岗位 JD 里常出现
- 不懂 **TikTok 电商真实数据从哪来**（Creative Center、Shop 后台、第三方工具）

### 面试侧

- 容易用**技术细节**代替**产品叙事**（Engine 权重讲一堆，用户价值讲不清）
- 缺少 **2 分钟 / 5 分钟项目介绍** 的标准话术
- 对「如果让你做 AI 产品助理，第一天干什么」没有答案

### 包装侧

- README 已改善，但还缺：**产品一页纸、竞品表、PRD 摘要、演示脚本**
- 线上 Demo 依赖 `DEEPSEEK_API_KEY`，面试官打开可能 **AI 功能不可用**

---

## 1.3 面试官会认为你缺什么

1. **有没有做过需求从 0 到 1 的过程**（不是从 GitHub 模板到 Demo）
2. **懂不懂跨境卖家真实工作流**（刷 TikTok → 验证趋势 → 找货 → 测款 → 备货）
3. **懂不懂 AI 产品边界**（什么该用 LLM、什么该用规则、什么必须 RAG）
4. **能不能写 PRD / 画原型 / 组织评审**（助理岗日常）
5. **知不知道 Coze / Dify / 工作流** — 很多团队已在用
6. **能不能把 TrendScout 讲成「产品」而不是「课程项目」**

# 二、AI产品助理实习冲刺路线图（10 天）

> **原则：不堆 TrendScout 新功能。** 把已有项目**产品化、文档化、面试化**，并补 1 个 **Coze/Dify 工作流** 独立作品。

## 第一周（Day 1–5）：产品底座 + 竞品 + PRD

### Day 1 — 定位与用户

**学习目标：** 把 TrendScout 从「功能列表」变成「解决谁什么问题」

**必学知识**

- 用户 Persona、Jobs-to-be-Done（JTBD）
- 跨境卖家选品链路：发现 → 验证 → 测款 → 放量 → 清仓

**推荐资料**

- 《启示录》第 1–3 章（用户与价值）
- TikTok Shop 卖家大学（官方入门）
- 36氪 / 晚点 任意 2 篇「TikTok 跨境电商」案例

**实战任务**

- 写 **Product Brief（1 页）**：
  - 目标用户：TikTok Shop / 独立站 新手卖家
  - 核心痛点 3 条
  - TrendScout 解决什么、**不解决什么**
  - 与「直接问 ChatGPT」的差异

**面试准备**

- 背熟 **「用一句话介绍 TrendScout」**  
  例：*「帮跨境卖家把 Google Trends 信号和趋势生命周期预测，变成可执行的选品与备货建议，而不是只看热点词。」*

---

### Day 2 — 竞品分析

**学习目标：** 建立「市场地图」，知道 TrendScout 站位

**必学知识**

- 竞品分析框架：功能 / 数据源 / AI 深度 / 定价 / 目标用户

**推荐资料**

- 直接体验：Google Trends、Exploding Topics、Glimpse、卖家精灵类工具（选 2~3 个）
- 妙手 / 店小秘 产品页（看「选品」模块怎么描述）

**实战任务**

- 做 **竞品对比表（Excel/Notion）**，至少 5 列 × 5 行：

| 产品 | 数据源 | AI 能力 | 目标用户 | 定价 | TrendScout 差异 |
| --- | --- | --- | --- | --- | --- |

- 写 **300 字结论**：TrendScout 的差异化是「趋势生命周期 Engine + LLM 解读」，不是「又一个 Trends 查询器」

**面试准备**

- 准备回答：**「你的竞品是谁？你为什么还能做？」**

---

### Day 3 — PRD 撰写（核心日）

**学习目标：** 掌握 AI 产品助理最核心的交付物

**必学知识**

- PRD 结构：背景 / 目标 / 用户故事 / 功能 / 非功能 / 指标 / 风险 / 排期
- AI 产品 PRD 多一块：**Prompt 输入输出、Fallback、成本、延迟**

**推荐资料**

- 人人都是产品经理：PRD 模板
- 任意 1 篇「AI 功能 PRD」范文（搜：AI 功能 PRD 模板）

**实战任务**

- 为 TrendScout 写 **《AI 趋势预警 + 选品顾问》PRD（精简 8~12 页）**，必须包含：
  1. 用户故事（As a 卖家, I want…）
  2. Alert Engine 规则 + `/api/alert-analysis` 的产品关系（诚实写：规则触发 + LLM 增强）
  3. 成功指标：如「从看到趋势到输出选品建议 < 3 分钟」
  4. **明确不做**：真实 TikTok API、自动下单、财务预测

**面试准备**

- 带 PRD 进面试（PDF），说：**「这是我负责模块的需求文档」**

---

### Day 4 — 原型 + 用户路径

**学习目标：** 证明你会「设计」，不只是会写代码

**必学知识**

- 关键路径原型：Dashboard → Forecast → Consultant → Alert
- Figma 基础（线框即可）

**推荐资料**

- B 站：Figma 线框图 1 小时入门

**实战任务**

- Figma **3~5 屏线框**（可截图放 README / 作品集）
- 画 **用户流程图**（比 README 里 Mermaid 多一步：**用户输入 keyword**）
- 标注每屏：**用户目标 / 系统反馈 / AI 参与点**

**面试准备**

- 演示顺序脚本（5 分钟）：
  1. 痛点 → 2. Dashboard → 3. Forecast+AI → 4. Consultant → 5. 预警 → 6. 局限与 Roadmap

---

### Day 5 — AI 工作流（Coze 或 Dify）

**学习目标：** 对齐 JD 里的 Coze / Dify / 工作流设计

**必学知识**

- Workflow vs Agent vs 单次 Chat
- 节点：输入 → 检索/工具 → LLM → 结构化输出

**推荐资料**

- Coze 官方文档 / Dify 官方文档（任选其一，跟到底）
- 看 1 个「电商选品 Bot」公开案例

**实战任务**

- 在 **Coze 或 Dify** 搭建 **「TrendScout 选品顾问 Lite」**（独立链接，不改代码）：
  - 输入：趋势名 + 阶段 + 评分
  - 输出：机会 / 风险 / SKU 建议 / 行动建议（JSON 或固定格式）
- 写 **《工作流设计说明》1 页**：为什么用工作流而不是裸 Chat

**面试准备**

- 回答：**「TrendScout 代码版 vs Coze 工作流版，各自适用场景？」**  
  代码版：可集成、可测试、可部署；Coze 版：快速验证、运营可改 Prompt

## 第二周（Day 6–10）：AI 深度 + TikTok + 面试冲刺

### Day 6 — Prompt Engineering + AI 产品设计

**学习目标：** 从「调了 API」升级到「设计了 AI 功能」

**必学知识**

- System / User Prompt、JSON 输出、Few-shot、Temperature
- Fallback、缓存、成本（单次 Forecast 分析 ≈ 3 次 LLM 调用）

**推荐资料**

- DeepSeek 官方 Prompt 指南
- 对照项目里的 `src/services/ai/prompts.ts`、`src/services/ai/trend-analyst.ts`

**实战任务**

- 写 **《TrendScout AI 设计文档》**（3~5 页）：
  - 4 个 Agent 角色表（Analyst / Consultant / Alert / Stream）
  - 每个的 **输入上下文、输出 Schema、Fallback 策略**
  - 诚实写：**Stream 分析缺数据 grounding 的风险 + 改进方案**（写进 Roadmap，不必开发）

**面试准备**

- 必答题：**「你怎么控制幻觉？」「AI 挂了怎么办？」**

### Day 7 — RAG / Agent 概念（应该会，不做大开发）

**学习目标：** 面试能讲清楚，能手画架构

**必学知识**

**推荐资料**

- 写 **200 字**：为什么 MVP 没用 RAG（数据量小、Mock 阶段）— 体现产品判断

**面试准备**

- 回答：**「如果让你做第二期，第一个加 RAG 的场景是什么？」**

# 三、面试知识树

| 领域 | 必会 ★ | 应该会 ○ | 加分项 △ |
| --- | --- | --- | --- |
| **AI 基础** | ★ AI 能做什么/不能做什么；幻觉；成本与延迟 | ○ Token、上下文窗口 | △ 模型选型对比 |
| **LLM 基础** | ★ Chat vs Completion；结构化 JSON 输出 | ○ Temperature、Top-p | △ 微调 vs Prompt |
| **Prompt Engineering** | ★ System/User；Few-shot；输出格式约束 | ○ Prompt 版本管理 | △ 自动评测 |
| **RAG** | ★ 概念、适用场景、与 Fine-tuning 区别 | ○ 向量库、Chunk、检索流程 | △ Hybrid Search |
| **Agent** | ★ Agent vs Workflow；工具调用概念 | ○ ReAct、多步规划 | △ Multi-Agent |
| **Workflow** | ★ 节点编排；分支；人工审核节点 | ○ 错误重试、超时 | △ 与业务系统集成 |
| **Dify** | ★ 搭通过 1 个工作流；知悉 Knowledge Base | ○ API 发布、日志 | △ 自托管 |
| **Coze** | ★ 搭通过 1 个 Bot/工作流 | ○ 插件、发布渠道 | △ 与飞书/微信集成 |
| **PRD** | ★ 结构完整；用户故事；验收标准；AI 非功能需求 | ○ 优先级 RICE/ICE | △ 数据埋点方案 |
| **用户访谈** | ★ 提纲；开放题；Insight 提炼 | ○ Affinity Mapping | △ 8+ 真实卖家样本 |
| **竞品分析** | ★ 5 竞品表格；差异化结论 | ○ 战略层对比 | △ 定价模型分析 |
| **TikTok 电商** | ★ Shop 基本链路；内容与选品关系 | ○ Creative Center；热卖趋势 | △ 广告 AI 应用 |
| **跨境电商** | ★ 选品→测款→备货；主要平台差异 | ○ 物流/合规概念 | △ 多站点策略 |

### 结合 TrendScout 的背诵锚点

| 知识点 | 对应项目证据 |
| --- | --- |
| Prompt ★ | `src/services/ai/prompts.ts` + Zod Schema |
| Workflow ★ | Forecast Engine → LLM → Dashboard 分层 |
| RAG ○ | 「第二期接卖家文档库，MVP 不需要」 |
| Agent ○ | 「Consultant 可演进为多步 Agent，当前是单次 JSON 报告」 |

- 岗位关键词 Coze/Dify/访谈/PRD **简历上无证据**

**建议：** 先别投大厂，先完成 10 天冲刺里的 **Brief + 竞品 + PRD**。

# 五、针对 TrendScout AI 的三条硬建议

> **原则：不鼓励继续堆功能。**

### 1. 简历项目描述改成产品语言

不要写「实现 8 个 API」；写：

> 「定义跨境卖家趋势选品流程，设计 Forecast/Alert 规则引擎与 DeepSeek 解读层，输出 PRD 与 41 条 Engine 单测保障逻辑正确性。」

### 2. 面试主动暴露边界

「当前 Mock 数据 + Google Trends，MVP 验证产品假设；第二期接 TikTok 信号与 RAG」— 面试官会认为你有产品成熟度。

### 3. 10 天内最高 ROI 的一件事

不是第 42 个测试，而是 **《AI 趋势预警 + 选品顾问 PRD》+ Coze 工作流链接** — 直接对齐 JD。
